# 星际商店看板娘配置说明

## 图片路径约定

1. 图片必须放在 `Textures/看板娘/` 目录下。
2. `<texturePath>` 字段中不写 `Textures/` 前缀，也不写文件扩展名。
3. RimWorld 会自动查找 `.png` / `.jpg` 格式。

### 示例

若贴图文件为：

```
星际商店文本版/Textures/看板娘/starstore_mascot.png
星际商店文本版/Textures/看板娘/猫娘.png
```

则 XML 中对应写法为：

```xml
<texturePath>看板娘/starstore_mascot</texturePath>
<texturePath>看板娘/猫娘</texturePath>
```

> RimWorld 内部使用正斜杠 `/` 作为路径分隔符，不需要写 `Textures/` 前缀。

### 推荐尺寸

- 512×512 或 1024×1024
- 尽量为正方形
- UI 中会按 `ScaleMode.ScaleToFit` 缩放，保持比例

## 单个看板娘 Def

每个看板娘使用独立的 `StarStore_MascotDef`：

```xml
<StarStore_MascotDef>
    <defName>StarStore_Mascot_Default</defName>
    <label>默认看板娘</label>

    <!-- 贴图路径：相对于 Textures/，不带扩展名 -->
    <texturePath>看板娘/starstore_mascot</texturePath>

    <!-- 该看板娘独立的问候语池，每次打开商店随机选一句 -->
    <greetings>
        <li>欢迎光临星际商店！</li>
        <li>今天需要点什么？</li>
        <li>新货已到，进来看看吧~</li>
    </greetings>
</StarStore_MascotDef>
```

## 侧边栏配置中的多看板娘随机

在 `StarStore_SidebarConfigDef` 中通过 `mascotDefNames` 指定多个看板娘：

```xml
<StarStore_SidebarConfigDef>
    <defName>StarStore_DefaultSidebar</defName>

    <!-- 多个看板娘 defName 列表：每次打开商店随机选一位 -->
    <mascotDefNames>
        <li>StarStore_Mascot_Default</li>
        <li>StarStore_Mascot_Robot</li>
        <li>StarStore_Mascot_Catgirl</li>
    </mascotDefNames>

    <!-- 未配置看板娘 Def 时的兼容问候池 -->
    <greetings>
        <li>欢迎光临星际商店，今天需要些什么？</li>
    </greetings>
</StarStore_SidebarConfigDef>
```

## Defs 文件夹结构

```
Defs/
├── 看板娘_MascotDefs/      ← 看板娘定义
├── 侧边栏_SidebarConfigDefs/  ← 侧边栏配置
├── 礼包_BundleDefs/        ← 礼包定义
├── 抽卡_GachaDefs/         ← 抽卡卡池
├── 稀有度_GachaRarityDefs/  ← 抽卡稀有度
├── 主按钮_MainButtonDefs/  ← 主按钮
├── 物品分类_ThingCategoryDefs/ ← 物品分类
├── 交易条件_TradeConditionDefs/ ← 交易条件
├── 商人_TraderKindDefs/    ← 商人定义
└── 通用_DefTypes/          ← 通用类型定义
```

## 运行机制

1. 商店 `PreOpen` 时，从 `mascotDefNames` 中随机选择一位看板娘。
2. 若该看板娘的 `greetings` 非空，则从其问候语池中随机选取一句显示。
3. 若未配置 `mascotDefNames`，则回退到旧字段 `mascotTexturePath` + `greetings`。
4. 看板绘制时，优先使用当前随机到的看板娘贴图；若不存在，则使用侧边栏配置的兼容贴图。

## 常见问题

### 文件名是否必须与 defName 一致？

不需要。只要 `<texturePath>` 与实际文件路径（相对于 `Textures/`）匹配即可。

### 出现 `BadTex`（紫黑格子）怎么办？

- 检查图片是否确实存在于 `Textures/<texturePath>.png`（或 `.jpg`）。
- 检查 `<texturePath>` 是否误写了 `Textures/` 前缀或文件扩展名。
- 检查图片尺寸是否过大（建议不超过 2048×2048）。

### 如何添加新的看板娘？

1. 准备一张正方形贴图，放入 `Textures/看板娘/` 目录。
2. 在 `Defs/看板娘_MascotDefs/星际商店看板娘.xml` 中新增一个 `StarStore_MascotDef`。
3. 将其 `defName` 添加到 `Defs/侧边栏_SidebarConfigDefs/星际商店侧边栏配置.xml` 的 `mascotDefNames` 列表中。
