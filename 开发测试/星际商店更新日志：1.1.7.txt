星际商店 v1.1.7 更新日志 2026-07-16

本次更新是一次大规模修复与完善，重点解决了多语言翻译缺失、出售模式 bug、以及界面 UI 问题。感谢所有反馈问题的玩家！

多语言翻译完善
━━━━━━━━━━━━━━━━━━━━

新增繁体中文支持
-四种语言完整覆盖：简体中文、英文、日文、繁体中文

DefInjected 翻译系统
-采用 RimWorld 原生 DefInjected 方案，将看板娘问候语、新闻公告、背景故事、侧边栏标题等 Def 字段的多语言翻译独立到语言文件中
-XML 配置文件保持中文明文，方便玩家自定义内容

商店入口按钮
-底部栏商店按钮的悬停文本现在会跟随游戏语言切换

礼包与抽卡
-礼包名称、描述（如"新手开拓包"）
-抽卡卡池名称、描述（如"基础补给箱"）
-稀有度名称（如"普通""精良""稀有""史诗""传说"）
-十连折扣标签、单抽/十连价格按钮文本
以上内容均已完成英/日/繁中翻译

C# 代码硬编码中文
-将代码中大量硬编码中文字符串改为翻译键调用（.Translate()）
-UI 标签如"每日折扣""星际新闻""背景故事""购买""卖出""清空"等全部支持多语言

日文翻译修正
-修正日文翻译中误用的简体中文词汇

Bug 修复
━━━━━━━━━━━━━━━━━━━━

出售模式"库存不足"（严重）
-修复所有有品质/材料变体的物品（武器、服装等）在卖出时提示"库存不足"的问题
-根因：网格上选的物品 key 为（def, null, null），但库存物品有实际品质/材料，严格匹配返回 false 导致找不到库存
-新增"宽松匹配"逻辑，任一方品质/材料为 null 时视为通配符

动物无法出售
-修复动物在卖出模式下无法出售的问题
-根因与机械族相同：动物是 Pawn，被库存映射的 category==Item 过滤排除
->> 按钮、卖出执行、仅显示库存过滤均已修复

机械族 >> 按钮返回 0
-修复卖出模式下机械族的 >> 按钮点击后数量显示为 0 的问题
-根因：库存映射只收集 ThingCategory.Item，Pawn 类型不在其中

输入框光标乱跳
-修复数量输入框在输入时光标位置乱跳的问题
-根因：原代码每帧重建 buffer 字符串传给 TextFieldNumeric，导致光标位置丢失
-改用持久化字典存储每个物品的输入文本缓冲

SplitOff 物品泄漏
-修复卖出时 SplitOff 返回的分离物品未被销毁的问题

空购物车静默返回
-修复点击交易按钮时购物车为空却没有任何提示的问题，现在会提示"购物车为空"

━━━━━━━━━━━━━━━━━━━━
界面 UI 修复
━━━━━━━━━━━━━━━━━━━━

品质按钮显示异常
-修复非中文语言下品质按钮显示为"<col"的问题
-根因：品质标签 GetLabel() 返回带 rich text 颜色标签的文本，截取前 4 字符破坏了标签

动物/机械族库存数量位置
-修复卖出模式下动物和机械族的库存数量显示位置与其他物品不一致的问题
-现在统一显示在价格下方

分类标签文字换行跳动
-修复拖动商店面板时分类标签（如"Mechanoids"）在两行之间来回跳动的问题
-根因：英文翻译文字宽度接近标签 Rect 宽度，亚像素波动导致换行/不换行来回切换
-关闭分类标签自动换行

看板新闻换行跳动
-修复拖动面板时看板新闻内容换行位置来回变化的问题
-文本宽度取整避免亚像素波动
