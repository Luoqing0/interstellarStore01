## 提示词

```
我正在开发一个 RimWorld 1.6 模组"星际商店"，请帮我继续开发。

### 项目位置
- 工作目录: c:\Steam\steamapps\common\RimWorld\Mods\星际商店文本版
- GitHub: https://github.com/Luoqing0/interstellarStore01

### 快速上手步骤
1. 读取项目状态文档: 开发测试/项目状态.md
2. 读取修改历史: 开发测试/修改记录/README.md
3. 读取主窗口代码: Source/MainTabWindow_星际商店.cs
4. 运行编译验证: cd Source; dotnet build 星际商店.csproj --configuration Release

### 核心功能
- 购买/卖出模式的商店界面
- 三种布局（大/中/小）
- 分类筛选、搜索、收藏
- 购物车、品质/材料选择

### 当前待实现
- 购买改为运输舱生成（优先轨道交易信标位置）

### 开发规范
- 使用 Conventional Commits 格式提交
- 每次修改后更新修改记录
- 编译验证后再提交到 GitHub

### RimWorld 原版代码参考
- 分类定义: c:\Steam\steamapps\common\RimWorld\Data\Core\Defs\ThingCategoryDefs\ThingCategories.xml
- 物品定义: c:\Steam\steamapps\common\RimWorld\Data\Core\Defs\ThingDefs_Misc\

### 报错处理
如果报错 "missing packageId"，检查是否有其他同名文件夹（星际商店/、星际商店备份/），当前版本 About.xml 是正确的。

请先读取项目状态文档了解项目结构，然后告诉我你理解了项目，准备开始工作。
```

## 补充说明

### rimworld-modding skill
这个 skill 主要提供：
- RimWorld Mod 开发流程指南
- XML Def 系统说明
- C# 开发最佳实践
- Harmony 补丁指南
- Steam Workshop 发布流程

它**没有 MCP 查询功能**，主要是提供开发指南。

### 新会话建议工作流程

1. **先读取文档**
   - `开发测试/项目状态.md` - 项目概述
   - `开发测试/修改记录/README.md` - 修改历史

2. **理解代码结构**
   - `Source/MainTabWindow_星际商店.cs` - 主窗口（partial class）

3. **编译验证**
   - 运行 `dotnet build` 确保代码正确

4. **开始工作**
   - 根据用户需求修改代码
   - 编译验证
   - 更新修改记录
   - 提交到 GitHub

## 常见问题

### Q: 新会话不懂 RimWorld API？
A: 让新会话使用 rimworld-modding skill，或者搜索原版代码：
```bash
# 搜索 ThingCategoryDef 定义
grep -r "ThingCategoryDef" c:\Steam\steamapps\common\RimWorld\Data\Core\Defs\

# 搜索特定类用法
grep -r "Widgets.ThingIcon" c:\Steam\steamapps\common\RimWorld\Data\Core\
```

### Q: 新会话不知道项目结构？
A: 让新会话读取 `开发测试/项目状态.md`，里面有完整的文件结构说明。

### Q: 新会话不知道当前问题？
A: 让新会话读取 `开发测试/项目状态.md` 的"当前问题状态"部分，里面有已解决和待实现的问题列表。
