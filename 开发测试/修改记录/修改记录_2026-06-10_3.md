## 修改详情

### 1. 购买逻辑 - 运输舱生成

**根因**: 项目状态文档要求"购买改为运输舱生成（优先轨道交易信标位置）"，当前使用 `GenPlace.TryPlaceThing` 直接放置物品，体验不够沉浸。

**修复**: 改用 `DropPodUtility.DropThingsNear` 通过运输舱降落生成物品。

**位置**: `Source/MainTabWindow_星际商店_交易.cs` 第 408-412 行

**修改前**:
```csharp
// 使用 GenPlace 直接生成物品到交易降落点
int 生成失败数 = 0;
IntVec3 dropSpot = 获取有效降落点;
for (int i = 0; i < 待生成.Count; i++)
{
    bool placed = GenPlace.TryPlaceThing(待生成[i], dropSpot, map, ThingPlaceMode.Near);
    if (!placed)
    {
        生成失败数++;
        Log.Warning($"星际商店: 购买物品 {待生成[i].def.defName} 放置失败 (位置 {dropSpot})");
    }
}
```

**修改后**:
```csharp
// AI 辅助生成：使用运输舱生成物品（优先轨道交易信标位置）
IntVec3 dropSpot = 获取有效降落点;
DropPodUtility.DropThingsNear(dropSpot, map, 待生成);
```

### 2. 卖出逻辑 - 运输舱生成白银收益

**根因**: 与购买逻辑保持一致，使用运输舱生成白银收益更有沉浸感。

**修复**: 改用 `DropPodUtility.DropThingsNear` 通过运输舱降落生成白银。

**位置**: `Source/MainTabWindow_星际商店_交易.cs` 第 472-474 行

**修改前**:
```csharp
bool placed = GenPlace.TryPlaceThing(silver, dropSpot, map, ThingPlaceMode.Near);
if (!placed)
{
    Log.Error($"星际商店: 出售收益白银放置失败 (数量 {白银数量}, 位置 {dropSpot})");
    Messages.Message("StarStore_SaleSilverDropFail".Translate(白银数量), MessageTypeDefOf.RejectInput);
}
```

**修改后**:
```csharp
// AI 辅助生成：使用运输舱生成白银收益
DropPodUtility.DropThingsNear(dropSpot, map, new List<Thing> { silver });
```

### 3. API 变化说明

**问题**: RimWorld 1.6 中 `DropPodUtility.MakeDropPodAt` 的签名从 `(IntVec3, Map, Thing)` 变为 `(IntVec3, Map, ActiveTransporterInfo)`，直接传入 Thing 会编译失败。

**解决方案**: 使用 `DropPodUtility.DropThingsNear(IntVec3, Map, IEnumerable<Thing>)` 方法，该方法接受 Thing 列表，更适合批量生成物品的场景。

**参考**: `RimWorld/Data/Core/Source/RimWorld/Game/Storyteller/Incidents/Workers/Map/IncidentWorker_ResourcePodCrash.cs`
