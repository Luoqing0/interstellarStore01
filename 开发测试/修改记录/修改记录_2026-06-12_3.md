# 修改记录 2026-06-12 (第二轮)

## 问题 1：物品格数量控制功能恢复
**根因**：上次修复大布局溢出时，将完整的滑块+输入框+4按钮替换为仅3元素的紧凑控件（[-][数量][+]），丢失了 `<<` 归零和 `>>` 最大功能。
**修复**：
- 新增 `绘制紧凑数量控制V2`：大布局单行 5 元素 `[<<][-1][N][+1][>>]`，20px 按钮 + 32px 可编辑输入框，总宽 ~130px 在 ~150px 格子内完全不溢出
- 中布局新增 `绘制完整数量控制`：紧凑滑块(14f) + 单行5元素按钮，恢复原版全部功能
- 中/小布局同样恢复出售变体选择按钮

## 问题 2：出售变体选择功能恢复
**根因**：上次修复物品网格时，大布局路径只绘制了紧凑数量控件，`物品有变体` 和 `Dialog_SellVariant` 的入口被跳过。
**修复**：大/中/小三种布局在 `!是购买模式 && 物品有变体(def)` 时均绘制"选择变体"按钮 + 数量控件，两小行紧凑排列。

## 问题 3：折扣刷新后区域消失
**根因**：`折扣随机种子 = Rand.Range(0, 99999)` 后调用 `获取今日折扣物品(折扣随机种子)`，该方法对某些随机值返回 null，导致整个折扣 if 块跳过。
**修复**：
- 新增 `上次折扣物品` 字段，每次显示折扣时记住
- 刷新后如果新随机种子返回 null，保留上次的折扣物品不消失
- 双重保底：先用之前缓存的，确保折扣区永不消失

## 问题 4：购物车标题英文乱码
**根因**：`"[购买] " + "StarStore_BuyList".Translate()` 拼合了硬编码中文前缀和翻译键，翻译键本身可能返回英文 fallback，导致 `[购买]BuyList` 样式的乱码。
**修复**：改为纯硬编码中文 `"购买清单"` 和 `"出售清单"`，不再依赖翻译键。

## 问题 5：Esc 先关看板再关商店
**根因**：看板和主商店是两个独立 `Window`，RimWorld 的 WindowStack 从栈顶开始分发 Esc，看板栈序在上面，所以先收到 Esc 并关闭自己，然后才轮到商店。
**修复**：
- `Window_看板` 设置 `closeOnCancel = false`，不响应 Window 默认的 Esc 关闭
- 重写 `OnCancelKeyPressed()`，内部调用 `主窗口.Close(false)` 直接关闭主窗口
- 主窗口 `PreClose` 中已有关闭看板的逻辑，效果：Esc → 主窗口关闭 → 看板随之关闭，一步到位

## 问题 6：物品格右侧空白填充
**根因**：固定列数的网格总宽可能小于可用区域宽度，多余空白全堆在右侧与购物车之间。
**修复**：`绘制物品网格` 中计算 `内容偏移X = Mathf.Max(0f, (rect.width - 内容宽) / 2f)`，每个格子 x 坐标加此偏移，内容居中显示，空白均匀分布在左右两侧。
