# 修改记录 2026-06-13

## 修改 1：室内/小型地图物资投放修复

**文件**：`MainTabWindow_星际商店_交易.cs`

**问题**：在房车等全室内地图中，`DropPodUtility.DropThingsNear` 找不到有效降落点时崩溃。

**修复**：两级投放策略。
- `获取有效降落点(Map map)`：简化逻辑——优先贸易信标→随机边缘→返回 Invalid
- `获取室内生成点(Map map)`：优先选中殖民者位置→地图中心→随机可通行格
- `执行购买()`：先尝试 `DropThingsNear`，若结果为空或坐标无效则调用 `GenPlace.TryPlaceThing` 在室内生成点逐个放置
- `执行卖出()`：同上两级策略

## 修改 5：def.label 空值保护（修复搜索/分类/打开商店时 NRE）

**文件**：`MainTabWindow_星际商店_交易.cs`

**问题**：部分 mod 注册了 `label == null` 的 ThingDef，`刷新物品列表` 遍历时 `def.label.ToLower()` 触发 `NullReferenceException`，导致商店无法打开、搜索框输入崩溃、分类切换崩溃、Dialog_SellVariant 崩溃。

**修复**：第 149 行增加 null 检查：`def.label == null || !def.label.ToLower().Contains(...)`

## 修改 6：室内地图落点检测改用 Roofed（修复运输舱越界）

**文件**：`MainTabWindow_星际商店_交易.cs`

**问题**：上一轮用坐标范围判断室内地图（`dropSpot.x >= 0 && dropSpot.z >= 0`），但 `RandomDropSpot` 在全室内地图仍会返回合法坐标（如 (10,0,5)），该坐标在屋顶下，`DropThingsNear` 仍会失败并将运输舱刷出界外。

**修复**：`执行购买()` 和 `执行卖出()` 中改为 `!dropSpot.Roofed(map)` 判断。全室内地图所有坐标有屋顶 → `可降落 = false` → 正确走 `GenPlace.TryPlaceThing` 室内生成。
