## 修改详情

### 1. TPS 性能下降（999→300）

**根因**: `获取库存映射` 每帧遍历 `map.listerThings.AllThings`，包含建筑、植物、投射物等所有 Thing。大型存档（大量 mod）下 AllThings 可达数千个，每个 Thing 执行 `TraderCanSell()` + LINQ `.Any()` 过滤，每帧开销显著。`绘制出售变体摘要` 中每个格子每帧执行 LINQ `GroupBy` + `Sum` + `Take(3)`，产生大量 GC 压力。

**修复**:
- `获取库存映射`：第一道过滤改为 `thing.def.category != ThingCategory.Item`（快速跳过建筑/植物/Filth 等），Chunks 检查用 for 循环替代 LINQ `.Any()`
- `绘制出售变体摘要`：手动遍历分组替代 LINQ `GroupBy`，避免匿名对象和闭包开销

**位置**:
- `Source/MainTabWindow_星际商店.cs` `获取库存映射()` 方法
- `Source/MainTabWindow_星际商店_物品网格.cs` `绘制出售变体摘要()` 方法

**验证**: 通过 MCP 查原版源码确认 `ThingCategory` 枚举值（Item=2, Building=3, Plant=4），`TraderCanSell` 实现简单但 `AllThings` 遍历量大才是瓶颈。

---

### 2. 无法出售鱼/食物类物品（有库存却提示 insufficient stock）

**根因**: `获取库存映射` 中用 `thing.def.tradeability.TraderCanSell()` 过滤物品。通过 MCP 查原版源码确认 `TraderCanSell()` 实现为 `tradeability == All || tradeability == Buyable`，**不包含 `Sellable`**。鱼的 `tradeability` 是 `Sellable`（MCP `get_def_details("Fish_Salmon")` 确认），被过滤排除。

完整 bug 链：
1. `刷新物品列表` 正确包含 `Sellable` 物品（卖出模式过滤条件含 `Sellable`）→ 网格显示鱼/食物
2. `获取库存映射` 用 `TraderCanSell()` 排除 `Sellable` → 库存映射中找不到
3. `执行卖出` 从库存映射找不到物品 → 提示 "insufficient stock"

**修复**: 把 `TraderCanSell()` 替换为直接枚举判断，包含 `Sellable`：
```csharp
Tradeability 交易性 = thing.def.tradeability;
if (交易性 != Tradeability.All && 交易性 != Tradeability.Buyable && 交易性 != Tradeability.Sellable) continue;
```

**位置**: `Source/MainTabWindow_星际商店.cs` `获取库存映射()` 方法

### 3. 高分屏 UI 缩放（2560×1600 + 1.5x 看板显示不全）

**根因**: 主窗口 1350f 宽，单独居中计算 `x = (UI.screenWidth - 1350) / 2`。看板窗口在主窗口左侧 `x = 主窗口x - 224`。2560×1600 + 1.5x 缩放下 `UI.screenWidth ≈ 1707`，主窗口居中 `x ≈ 178`，看板 `x = 178 - 224 = -46`，**超出屏幕左边被裁剪**。

**修复**: 主窗口+看板作为整体居中计算，加保底防止超出屏幕：
```csharp
float 整体宽 = windowRect.width + 看板宽 + 4f;
windowRect.x = (UI.screenWidth - 整体宽) / 2f;
if (windowRect.x < 看板宽 + 4f) windowRect.x = 看板宽 + 4f;
if (windowRect.x + windowRect.width > UI.screenWidth)
    windowRect.x = UI.screenWidth - windowRect.width;
```

**位置**: `Source/MainTabWindow_星际商店.cs` `PostOpen()` 方法
