## 记录格式规范

每个修改记录文件包含以下部分：

1. **基本信息**: 日期、涉及文件数、工具/模型、对应报错
2. **修改详情**: 按模块分组，每组包含：
   - **根因**: 问题原因分析
   - **修复**: 具体修改内容
   - **位置**: 修改的代码位置
3. **修改文件清单**: 表格汇总所有修改的文件
4. **编译结果**: 编译验证状态
