## 修复清单

### 问题1: 礼包/抽卡界面文本未翻译
**根因**: 礼包(BundleDef)、抽卡(GachaDef)、稀有度(GachaRarityDef)的 label/description/badgeText 等字段是中文明文 Def 字段，没有 DefInjected 翻译。

**修复**: 创建 9 个 DefInjected 翻译文件（英/日/繁中 × 3 种 Def 类型），共 93 条翻译。

| Def 类型 | 字段 | 每语言条目数 |
|---------|------|:-----------:|
| StarStore_BundleDef (3个) | label, description | 6 |
| StarStore_GachaDef (3个) | label, description, tenPullDiscountLabel, singleLabel, tenLabel | 15 |
| StarStore_GachaRarityDef (5个) | label, badgeText | 10 |

### 问题2: 品质按钮显示 `<col`
**根因**: `QualityCategory.GetLabel()` 返回带 rich text 颜色标签的文本（如 `<color=#xxx>Normal</color>`），截取前 4 个字符变成 `<col`。

**修复**: 用 `.StripTags()` 去掉颜色标签后再截断。
- 文件: `MainTabWindow_星际商店_物品网格.cs` 第 221 行

### 问题3: 动物/机械族库存数量位置不一致
**根因**: `绘制出售变体摘要` 用 `获取库存映射` 查库存，Pawn 不在库存映射中，直接 return 不显示库存信息。动物/机械族的库存数量显示在了其他位置（`>>` 按钮旁），与普通物品位置不一致。

**修复**: 在 `绘制出售变体摘要` 开头加 Pawn 独立路径，用 `机械族管理器.获取殖民地机械族/动物` 查数量，显示在价格下方（与普通物品一致）。
- 文件: `MainTabWindow_星际商店_物品网格.cs` 第 528-551 行

### 问题4: 分类标签文字换行跳动
**根因**: `Widgets.Label` 默认开启自动换行，当英文翻译文字宽度接近 Rect 宽度时，窗口拖动导致的亚像素变化让文字在换行/不换行间来回切换。

**修复**:
1. 分类标签: 绘制前 `Text.WordWrap = false`，绘制后恢复
2. 看板新闻: 文本宽度用 `Mathf.Floor()` 取整，避免亚像素波动
- 文件: `MainTabWindow_星际商店_标题栏.cs` 第 150-155 行, `MainTabWindow_星际商店_看板.cs` 第 130 行
