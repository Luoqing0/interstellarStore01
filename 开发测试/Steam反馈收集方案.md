# 星际商店 - Steam 玩家反馈收集方案

---

## 一、Steam 创意工坊自带的反馈渠道

| 渠道 | 用途 | 你的操作 |
|------|------|---------|
| **评论区 (Comments)** | 玩家随手留言、评分、提问 | 定期回复，置顶一条反馈引导帖 |
| **讨论区 (Discussions)** | 分类讨论 Bug、建议、展示、综合 | 主动创建几个板块帖子 |
| **评分/点赞** | 判断模组受欢迎程度 | 根据好评差评调整优化方向 |
| **Bug 报告工单** | 无，Steam 创意工坊没有专门 Bug 系统 | 需要在讨论区自建模板 |

---

## 二、推荐布局

### 1. 置顶一条「反馈引导帖」

在讨论区置顶如下内容：

```
标题：如何提交 Bug / 建议 / 平衡性反馈

欢迎各位使用星际商店！

提交反馈前，建议附上 Player.log 文件：
Windows: %USERPROFILE%\AppData\LocalLow\Ludeon Studios\RimWorld by Ludeon Studios\Player.log

反馈格式：
- 问题类型：Bug / 建议 / 平衡性 / 其他
- RimWorld 版本：
- 问题描述：
- 重现步骤：
- 截图/日志：

也可以加入 QQ 群参与内测讨论：[你的群号或群链接]
```

### 2. 创建讨论区分区帖子

建议在讨论区创建：
- 【Bug 汇总】已知问题与修复进度
- 【建议收集】功能与体验建议
- 【兼容性报告】与其他模组一起使用的情况
- 【使用技巧】常见问题解答（FAQ）

---

## 三、提高反馈质量的技巧

| 问题 | 解决方法 |
|------|---------|
| 玩家只说"崩溃了" | 置顶帖明确列出 Player.log 路径和提交格式 |
| 反馈散落在评论区 | 每次回复时复制「请到讨论区按格式发帖」 |
| 重复问题多 | 定期更新「已知问题」主帖，让玩家先看 |
| 语言混杂 | 中英文分别开帖，或在主帖中提供双语模板 |

---

## 四、外部反馈渠道（可选但推荐）

| 渠道 | 适合场景 | 注意 |
|------|---------|------|
| **QQ 群** | 中文核心玩家、快速沟通 | 适合热修复和内测迭代 |
| **Discord** | 海外玩家 | 如果希望扩展国际市场 |
| **GitHub Issues** | 技术型玩家、Bug 追踪 | 方便打标签、关联版本 |
| **腾讯文档/飞书表单** | 结构化收集 | 和 Steam 讨论区配合使用 |

---

## 五、反馈处理工作流

```
玩家反馈
   ↓
Steam 讨论区 / 评论区 / QQ 群
   ↓
每日/每两日集中查看一次
   ↓
分类：
  - 严重 Bug → 立刻复现 → 热修复
  - 一般 Bug → 记录到修复列表
  - 建议 → 评估可行性 → 加入待实现或婉拒
  - 兼容性 → 安装对应模组复现
   ↓
在置顶帖更新进度
   ↓
发布新版本 → 在更新日志中感谢反馈者
```

---

## 六、收集反馈时的心态建议

- 不要每条建议都接受，优先修影响面大的 Bug
- 对热心反馈的玩家公开致谢，能提高参与感
- 负面评价不要硬杠，表达「已记录，会考虑」即可
- 定期发布更新日志，让玩家看到模组在持续维护
